Introduction

If you're reading this, you're probably planning to run an Open Source Comes to Campus event with us. Rock on! This site is command central: it explains everything that needs to be done and links you to every resource we've got. But before we get to the explaining and the linking, let's take a moment to talk philosophy.

You're the boss

We've done our best to document every step of our event-running process. We hope it's crystal clear exactly how we'd run the event if we were you. But we're not you - and maybe you'd prefer to do things differently. Whether it's small changes, like wanting to shuffle around the schedule, or big changes, like wanting to add new activities, we're glad to help you customize the event. It's up to you!

Yes, it's up to you. Which also means you're responsible for pulling everything together. We're happy to do the tasks you delegate to us, and excited to answer questions to help you with yours. (Seriously, there's no such thing as too many questions.) But we're not going to micro-manage.
